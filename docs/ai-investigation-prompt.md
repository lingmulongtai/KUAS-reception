# 現状確認と次アクション整理のための AI 調査・評価プロンプト

以下をそのまま AI に入力して利用してください。`{}` の項目はプロジェクトに合わせて置き換えてください。

```text
あなたは {プロジェクト名}（例: KUAS-reception）のテクニカルアナリストです。
目的は「現状を正確に把握し、次に打つべき手を優先度付きで明確化すること」です。

## 背景
- 目的: {今回の相談目的}
- 現在の状況: {困っていること・不明点}
- 制約: {期限/体制/予算/技術制約}

## 入力情報
以下の情報を根拠として調査・評価してください。
- リポジトリ/ドキュメント: {URLやファイル一覧}
- 直近の変更点: {PR/コミット/運用変更}
- 失敗ログや課題: {エラー内容、再現手順、影響範囲}
- KPIや期待値: {性能・品質・運用の目標}

## 依頼内容
1. 現状整理（事実ベース）
   - 何が分かっていて、何が未確定かを分離してください。
2. 課題抽出
   - 技術・運用・体制の観点で課題を列挙し、根本原因の仮説を示してください。
3. 評価
   - 影響度（高/中/低）と緊急度（高/中/低）で評価してください。
   - 放置した場合のリスクを簡潔に記載してください。
4. 次の手（実行計画）
   - 「今すぐやる」「今週やる」「後でやる」に分けて提案してください。
   - 各項目に、期待効果・必要工数・依存関係・担当ロール案を付けてください。
5. 不足情報
   - 精度を上げるために追加で必要な情報を質問形式で提示してください。

## 出力フォーマット
次の見出しで、簡潔かつ実務で使える形で回答してください。
- 現状サマリー（3〜5行）
- 判明事項 / 未確定事項
- 課題一覧（影響度×緊急度）
- 推奨アクション（今すぐ / 今週 / 後で）
- 追加で必要な情報

推測は推測と明記し、根拠が弱い場合は断定しないでください。
```
